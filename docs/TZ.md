# ЕГЭ AI

**Аналитика спроса, конкурентов, уникальных фич и ТЗ на backend**
_Версия для обсуждения с разработчиком / партнёром_

> **Статус данных:** факты проверены по открытым источникам, продуктовые выводы отмечены как
> гипотезы. Точные частотности поисковых запросов требуют отдельной выгрузки из
> Wordstat / ASOmobile / Keys.so / Serpstat.

> **Главный вывод:** не конкурировать с онлайн-школами как «ещё один курс», а строить карманную
> ИИ-систему ежедневного ведения ученика до нужного балла:
> диагностика → план → задания → проверка → слабые темы → прогноз → перестройка плана.

## 2. Аналитика спроса: боли, сезонность, предметы, токены

### 2.1. Главные боли пользователя

| Боль ученика              | Что он хочет                              | Продуктовый ответ                                  |
| ------------------------- | ----------------------------------------- | -------------------------------------------------- |
| «Я с нуля»                | Понятный старт без стыда и сложной теории | Онбординг + диагностика + маршрут с базовых тем    |
| «Осталось мало времени»   | Понять, что реально успеть                | Режимы 7/30/90 дней и баллоёмкие темы              |
| «Нет денег на репетитора» | Дешёвый заменитель ежедневной поддержки   | ИИ-куратор, проверки, план и микрозадания          |
| «Не понимаю ошибки»       | Не просто ответ, а объяснение причины     | Разбор ошибки + похожие задания                    |
| «Нужен план»              | Каждый день знать, что делать             | Ежедневный план, который перестраивается по прогрессу |
| «Хочу понять баллы»       | Прогноз и понятную цель                   | Прогноз баллов диапазоном + темы для прироста      |

### 2.2. Сезонность

- Официальный основной период **ЕГЭ-2026 — с 1 июня по 9 июля**; пик спроса на подготовку логично ожидать до июня.
- **Панический спрос** — апрель–июнь: быстрые планы, проверка сочинений, пробники, «что учить в последний месяц».
- **Стратегический спрос** — сентябрь–февраль: курсы, годовые программы, системная подготовка.

### 2.3. Предметы с высоким потенциалом

| Предмет               | Почему важен                                              | Фичи для старта                                          |
| --------------------- | -------------------------------------------------------- | ------------------------------------------------------- |
| Русский язык          | Обязательный; высокая боль вокруг сочинения и критериев  | Проверка сочинения, карта ошибок, план улучшения        |
| Математика профиль    | Высокая тревога и спрос на пошаговые разборы             | Разбор задачи, подсказки, похожие задания (с ограничением уверенности ИИ) |
| Обществознание        | Один из самых массовых предметов по выбору              | Термины, теория простым языком, тесты, повторение       |
| Информатика           | Цифровая аудитория, совместимость с кодом и алгоритмами  | Проверка кода, объяснение ошибок, задачи по Python      |
| Биология/химия/физика | Значимый спрос, но сложнее контентная база               | Теория, карточки, тесты, разбор типовых задач           |

## 3. Аналитика предложения: конкуренты и их механики

> **Рыночная картина:** крупные конкуренты в основном продают образовательную систему (курсы,
> занятия, домашки, кураторы, пробники, платформы, иногда ИИ-помощники). Поэтому «у нас есть ИИ»
> уже не является уникальностью.

| Конкурент           | Что заявлено в открытых источниках                                                                 | Вывод для нас                                                              |
| ------------------- | -------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------- |
| Умскул              | Курсы ЕГЭ/ОГЭ; индивидуальная программа, домашки с проверкой, разбор ошибок, помощь куратора        | Сильный бренд и курсная модель. Отличаться не курсом, а ежедневным маршрутом |
| Фоксфорд            | Курсы ЕГЭ; автопроверка + письменные задания с экспертами; комплекты по кодификатору и пробники     | Сильная инфраструктура. Наша ниша — быстрый ИИ-помощник и низкий порог входа |
| Сотка               | Тарифы, пробные варианты, проверка наставником, выявление слабых тем, динамика результатов          | Уже закрывает пробники и слабые темы. Нужна связка «диагноз → перестройка плана» |
| Вебиум              | Подготовка с нуля, наставник, личный трек, домашки и пробники; сильная упаковка страхов ученика      | Хорошо работает с тревогой. Делать ещё проще и дешевле: карманный куратор  |
| ЕГЭLAND             | Уроки, домашки, личный наставник, ИИ-куратор, материалы и платформа                                 | ИИ уже заявлен → уникальность в глубине персонализации, «ОС подготовки»    |
| Яндекс Репетитор AI | Бесплатный AI по профильной математике: разбор решений, подсказки, обучен на примерах; «AI может ошибаться» | Сильный игрок в AI-математике. Не воевать по матем. напрямую; логичнее русский/сочинения + план |

## 4. Рыночная возможность и позиционирование

> **Главная гипотеза:** свободная ниша — не очередная онлайн-школа, а **ИИ-операционная система
> подготовки к ЕГЭ**, которая ведёт ученика каждый день.

| Не делаем                          | Делаем                                                             |
| ---------------------------------- | ----------------------------------------------------------------- |
| Не «ещё один курс с уроками»       | Ежедневный маршрут: что делать сегодня, чтобы добрать баллы        |
| Не «чат с нейросетью»              | ИИ-куратор, встроенный в план, задания, прогресс и пробники        |
| Не «решатель задач для списывания» | Разбор ошибки, подсказки и похожие задания                        |
| Не «база заданий»                  | Система диагностики, слабых тем, прогноза и персональных рекомендаций |

**Рабочее позиционирование:** «ИИ-тренер для ЕГЭ: найдёт слабые темы, составит план до экзамена,
проверит задания и каждый день будет вести к нужному баллу».

## 5. Фичи для отличия и роста спроса

| Фича                          | Суть                                                                              | Почему работает на спрос                              |
| ----------------------------- | --------------------------------------------------------------------------------- | ----------------------------------------------------- |
| Маршрут до балла              | Ученик вводит предмет, дату, уровень, цель, время → система строит маршрут         | Сильная боль «что делать каждый день»; рост удержания |
| Пробник с диагнозом           | После пробника — не только балл, но и причины потерь, слабые темы, план на 7 дней | Связка с перестройкой плана — ключевой UX             |
| Разбор ошибки, а не ответ     | ИИ объясняет конкретную ошибку, даёт похожие задания, обновляет карту слабых тем   | Снижает риск имиджа «сервис для списывания»           |
| Антипаника 7/30/90 дней       | Режимы под оставшееся время: 7/30/90 дней, «с нуля»                                | Сезонный спрос и рекламные креативы                   |
| Карта слабых тем              | Визуальная карта: зелёный/жёлтый/красный/серый                                    | Прогресс понятный и игровой                           |
| Проверка сочинения            | Проверка по критериям, примерный балл, ошибки, рекомендации, упражнения           | Самая понятная MVP-ценность для русского              |
| Родительский отчёт            | Недельный отчёт: занятия, темы, слабые места, прогноз, план                        | Часто платит родитель; продаём спокойствие и контроль |
| Поступление от баллов         | Куда примерно проходит, сколько не хватает, какой предмет выгоднее подтянуть       | Ценность за пределами «решать задания»                |
| ИИ-куратор дня                | Активные сообщения: пропуски, маленькое действие, возврат в план, снижение тревоги | Заменяет часть роли наставника дешевле и масштабируемее |

## 6. Приоритеты MVP

> **Рекомендация:** не делать сразу «все предметы». Самый реалистичный старт — **русский язык и
> проверка сочинений**: ценность ИИ видна сразу, риск технических ошибок ниже, чем в математике.

| Версия   | Что входит                                                                                                          | Цель                                            |
| -------- | ------------------------------------------------------------------------------------------------------------------ | ----------------------------------------------- |
| MVP 1.0  | Регистрация, профиль, русский язык, проверка сочинения, карта ошибок, план на 7 дней, лимиты ИИ, админка промптов, логирование стоимости | Проверить, платят ли за ИИ-проверку и план      |
| MVP 1.1  | Задания первой части, слабые темы, ежедневные задания, пробник по русскому                                          | Увеличить удержание и частоту использования     |
| MVP 2.0  | Математика/общество, фото задания, прогноз баллов, родительский отчёт                                               | Расширить LTV и платные тарифы                  |

## 7. Техническое задание на backend

### 7.1. Описание проекта

Backend для веб/мобильного приложения подготовки к ЕГЭ на основе ИИ. Приложение помогает ученику
выбрать предметы, пройти диагностику, получить персональный план, решать задания, проверять
ответы/сочинения/фото, видеть прогресс и прогноз баллов.

> **Главная логика backend:** не просто хранить задания, а **обновлять индивидуальный маршрут
> подготовки** на основе результатов ученика.

### 7.2. Роли пользователей

### 7.6. AI Orchestrator: требования

- Все ИИ-запросы логируются: пользователь, тип запроса, модель, вход/выход, токены, стоимость, статус, ошибка.
- Ответ ИИ возвращается в структурированном JSON, чтобы frontend стабильно показывал результат.
- Для дорогих задач (фото, сочинения, генерация плана) — очередь и статус обработки.
- Лимиты по тарифам: число ИИ-проверок в день, доступные типы проверок, лимит предметов.
- Для повторяемых объяснений — кеширование.
- ИИ не отдаёт только готовый ответ без объяснения: продуктовая логика — обучение и разбор ошибки.
- Для спорных ответов ИИ возвращает `confidence_score` и пометку «нужна перепроверка».

## 8. Организация модулей backend между собой

1. Пользователь регистрируется через **Auth Module**.
2. **User Profile Module** собирает предметы, цель, дату экзамена и доступное время.
3. **Study Plan Module** создаёт первичный план подготовки.
4. **Task Bank Module** отдаёт задания под текущий день и слабые темы.
5. **Answer Checking Module** принимает ответ ученика.
6. Простая проверка — автоматически; сложная — запрос уходит в **AI Orchestrator**.
7. **AI Orchestrator** возвращает структурированный результат: балл, ошибки, объяснение, рекомендации.
8. **Progress Module** обновляет статистику по теме.
9. **Recommendation Module** предлагает следующие действия.
10. **Score Forecast Module** пересчитывает прогноз баллов.
11. **Mock Exam Module** после пробника отдаёт результат и слабые темы.
12. **Study Plan Module** перестраивает план после пробника, пропусков или частых ошибок.
13. **Subscription Module** проверяет лимиты перед дорогими действиями.
14. **Analytics Module** фиксирует события для анализа продукта и экономики ИИ.

## 10. Риски, спорные места и что проверить до разработки

| Риск                            | Почему опасно                                              | Что делать                                                       |
| ------------------------------- | --------------------------------------------------------- | --------------------------------------------------------------- |
| ИИ ошибается                    | Особенно в математике, рукописных решениях, сложных критериях | `confidence_score`, дисклеймеры, гибридная проверка, тестовые наборы |
| Высокая стоимость ИИ            | Фото и сочинения могут съесть маржу подписки              | Лимиты, кеширование, разные модели под задачи, логирование токенов |
| Нет качественной базы заданий   | Без контента продукт не удержит пользователя              | Начать с одного предмета и легальных/собственных заданий         |
| Слишком широкий MVP             | Разработка затянется и не проверит гипотезу быстро        | MVP 1.0: русский + сочинение + план + прогресс                   |
| Сервис используют для списывания | Портит бренд и образовательную ценность                  | Давать подсказки, объяснения и похожие задания, а не просто ответ |
| Юридические вопросы с контентом | Нельзя бездумно копировать чужие задания и материалы      | Проверить права, использовать ФИПИ как структуру, создавать свои задания |

### 10.1. Что обязательно проверить цифрами

| Гипотеза                                    | Как проверить                                       |
| ------------------------------------------- | --------------------------------------------------- |
| Есть спрос на «проверку сочинения ЕГЭ»      | Wordstat/ASOmobile/Директ + лендинг + тест рекламы  |
| Школьник готов платить 100–300 ₽/мес        | Интервью + тест оплаты + pre-order                  |
| Родительский отчёт повышает оплату          | A/B тест: тариф без отчёта и тариф с отчётом         |
| Маршрут до балла удерживает                 | Метрика retention D7/D14 и выполнение плана          |
| ИИ-проверка достаточно точная               | Сравнить с экспертной проверкой на наборе сочинений  |
