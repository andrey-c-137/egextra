# Варианты экзаменов — папка для загрузки

Сюда вы выкладываете готовые варианты в `.txt`. Я (Claude) напишу импортер,
который разберёт эти файлы и **заменит демо-контент** (всё, что помечено `source: demo`)
на ваши реальные задания, темы и пробники.

## Куда класть файлы

Один **файл = один вариант** (= один пробник). Кладите в папку нужного экзамена:

```
content/variants/
  ege-russkiy/            ЕГЭ — Русский язык
  ege-matematika-profil/  ЕГЭ — Математика (профильная)
  ege-matematika-baza/    ЕГЭ — Математика (базовая)
  oge-russkiy/            ОГЭ — Русский язык
  oge-matematika/         ОГЭ — Математика
```

Имя файла: `variant-01.txt`, `variant-02.txt`, … (порядок = номер варианта).

## Формат файла

Сверху — шапка варианта (один раз), ниже — задания, разделённые строкой `=== ЗАДАНИЕ ===`.
Поля — это те же названия, что у вас в выгрузке (см. ваш скрин). Значение поля может быть
многострочным: оно продолжается до следующего известного поля или до `=== ЗАДАНИЕ ===`.

Используются не все поля — обязательные помечены **(*)**, остальные сохраню «про запас».

| Поле | Обяз. | Куда идёт |
| --- | --- | --- |
| `Экзамен` | * | ЕГЭ/ОГЭ (можно один раз в шапке) |
| `Предмет` | * | название предмета (в шапке) |
| `Год модели` |  | сохраню в метаданных |
| `Номер задания` | * | номер по официальной структуре |
| `Тема` | * | тема (по ней строится план и карта тем) |
| `Проверяемое умение` |  | сохраню |
| `Уровень сложности` | * | базовый/повышенный/высокий → 1/2/3 (важно для плана) |
| `Тип ответа` | * | краткий → автопроверка; развёрнутый → проверка ИИ |
| `Максимальный балл` | * | макс. балл задания |
| `Текст задания` | * | условие, которое видит ученик |
| `Правильный ответ` | * (для кратких) | эталон; несколько вариантов — через `;` |
| `Решение` |  | показываю в пояснении |
| `Критерии` |  | для развёрнутых (проверка ИИ) |
| `Типичные ошибки` |  | сохраню |
| `Статус проверки` |  | импортирую только `Принято` (черновики пропущу) |

### Соответствия

- **Уровень сложности:** `базовый` → 1, `повышенный` → 2, `высокий` → 3.
- **Тип ответа:** `краткий` / `выбор` / `соответствие` → автопроверка по эталону;
  `развёрнутый` → проверка ИИ по критериям (балл ставит модель).
- **Несколько допустимых ответов:** перечислите через `;` (например `2836;3628`).
- **Пробник** собирается автоматически из всех заданий файла (по возрастанию номера).

## Важно про автопроверку кратких ответов

Ответ ученика сверяется с эталоном без учёта регистра, пробелов и `ё/е`.
Если у задания несколько правильных форм — перечислите их через `;`.

## Как лучше передать

1. **Сначала закиньте ОДИН файл** (любой вариант) — я проверю, что парсер точно
   читает ваш формат, и при необходимости подстрою его под вашу реальную разметку.
2. После подтверждения — заливайте остальные, я импортирую все и удалю демо.

Пример разметки — в [`_TEMPLATE.txt`](./_TEMPLATE.txt).
